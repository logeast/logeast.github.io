svg 多线条动画做起来挺容易的，但也有一些需要留意的地方，这里记录一下。

接下来将实现一个 loading 图标，先看看最终效果。

再来个慢动作的。

分析一下图标动画，发现很简单，先是三条线依次出现，紧接着三条线再依次消失，循环往复。

1. 单个线条动画

首先我们要拿到 svg 图标的代码，在此基础上进行动画的添加。直接用编辑器打开就会看到源码长这样。

我们需要做的就是通过操作 path 元素来实现动画效果。svg 路径动画的原理其实就是通过改变线条的偏移量来欺骗视网膜从而实现动画效果，这里需要用到两个属性，简单提一下。

  1. stroke-dasharry: <dasharray> 指定短划线和缺口的长度。
  2. stroke-dashoffset 指定了 dash 模式到路径开始的距离，其实就是线条的偏移量。

2. 多个线条依次出现动画

最开始的的思路，给每个线条加动画，然后依次给线条延时，循环往复，从而实现依次出现的视觉效果。一点都难不倒我们。

然而，实现后的效果却长这样。忍不住念了两句诗，多情自古空余恨，好梦由来最易醒。

赶紧分析原因，发现就是“循环往复”这儿出了问题，各个线条各干各的，只是在第一次播放的时候有延时，以后就会争先恐后地跑出来，没有相互关联起来，我们设置延时的初衷没有达到啊。

现在我们是将动画分成了三个，才会出现上面的情况，知道了症结所在，就可以对症下药了。如果我们将他们合为一个，并将整个动画分割成三段，每段控制一个线条，岂不是就可以完美解决延时问题了？画张图来分析一下。

于是我们就可以很轻松写出三段动画了，首先分析每段线条动画所占的时间比例，它只在比例限定的时间里活跃，其他时间处于静止状态，这样我们也不需要设置延时了，我们将三段动画的时间比例分别定为 40%，20%，40%，来看看代码长什么样。

现在，想要的效果出现了，这就是柳暗花明又一村最佳表现啊。

依次出现的动画到此告一段落。
